# UNIT PIR —— 硬件 Ground Truth

本文件是 **Unit PIR** 这颗外设单元的板级硬件事实,可复制到任意挂载该单元的项目。**项目特定逻辑请勿写在这里**,放 `CLAUDE.md`。

来源:
- M5Stack 官方产品页 PDF(`PIR.pdf`,SKU **U004**)
- 板载芯片:**PIR_AS312**(热释电传感器)、**HT7533**(3.3V LDO)

---

## 1. 一句话定位

被动式热释电红外(PIR)运动探测器。检测到人体/物体红外变化时 **输出数字高电平**。**GPIO 单元(PORT.B)**,不是 I2C。

---

## 2. 接口与接线(HY2.0-4P / PORT.B)

| 颜色 | 信号 | GPIO 用途 |
|---|---|---|
| Black | GND | — |
| Red | 5V | 整板供电 5V |
| Yellow | **NC** | 不接 |
| White | **Digital Output** | **检测输出**,接主控一个可读 GPIO |

> ⚠️ **易错点(与 Unit RGB 相反!)**:PIR 的信号在 **White** 线;Yellow 是 NC。
> 而 Unit RGB 的信号在 **Yellow** 线、White 是 NC。两者接线方向相反,换单元时务必核对。

- 接 Core2 **PORT.B**:输出在 **White** 线 → 落 Core2 **G36**(`BSP_PIN_PORTB_ADC`,input-only,读数字高低电平正好)。Yellow(G26)不接。
- **驱动映射(本平台)**:组件 `components/units/unit_pir`;输入脚 = `BSP_PIN_PORTB_ADC`(G36);固件配输入 + 软件做低→高边沿检测当"有人事件"。
- ⚠️ 与 Unit RGB 恰好各占 PORT.B 一脚:RGB 用 Yellow/G26(输出)、PIR 用 White/G36(输入),互不冲突。

> 物理:32 x 24 x 12.5 mm,5.1 g,2×LEGO 兼容孔。

---

## 3. 行为 / 时序

- 检测到运动:输出脚 **拉高**。
- 延时:**固定 2 秒**(高电平至少维持 2s)。
- 触发模式:**可重复触发**——延时窗口内若持续检测到运动,会不断延长高电平时间。
- 检测距离:**500cm**;感应范围:**< 100°**(广角)。
- 软件侧通常做边沿检测(低→高)当作"有人事件",并自行做去抖/冷却。

---

## 4. 电源

- 输入 5V → 板载 **HT7533** 降 3V3 供 PIR。
- 输出电平为 **3.3V 逻辑**高/低。
- 静态电流:`< 60µA`。工作温度:**-20℃ ~ 80℃**。

---

## 5. 关键注意事项 / 易错点

- 信号在 **White** 线,不是 Yellow(见上方对比框)。
- 上电后 PIR 需要数十秒稳定期,刚上电的输出不可信,启动时先丢弃/屏蔽一段时间。
- 2 秒延时是硬件固定的,软件改不了"硬延时";要更短的事件粒度需在软件侧靠边沿判断。
- 对温度变化、空调出风、宠物等也会触发,属物理特性。

---

## 6. 没有的东西 / 常见误解

- **没有 I2C**,也不上报距离/角度;只有一根**数字高低电平**输出。
- 不能调灵敏度/延时(无引出的电位器寄存器)。
