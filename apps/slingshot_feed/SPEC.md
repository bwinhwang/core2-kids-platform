## 0. 立项依据(为什么是它)

对准 ROADMAP §0 归因四条 + 一条战略修复:

1. **实体感 = 摇杆拉-放的弹簧回路**:拉的阻尼 + 松手弹回,是可抓握的真实机械反馈(vs feed_monster
   隔空手远近"对空气"的类型级不匹配——ROADMAP 明确这条路不是调参能救的)。
2. **连续比例映射**:拉多远 → 飞多远(力度),拉的方向 → 发射角;**实时抛物线预览**让"拉多少
   打多远"当场可见(自教)。
3. **决策随技能成长**:瞄准是随练随稳的真技能(chain_lab 对齐、fish_pond 追踪已证瞄准对幼儿好玩);
   动物每轮位置变 → 每次都是新的瞄准决策。
4. **收集/庆祝元循环**:喂饱一只动物 → 蹦跳长大 → 换下一只;喂够一批 → 全场派对。

**战略价值**:喂食图式本身受欢迎(见什么喂什么是幼儿本能),feed_monster **只错在隔空输入**;
本作用可抓握摇杆把它复活。**新技能轴 = 拉-放弹道瞄准**(vs chain_lab 静态抓取直落、fish_pond
移动目标收放线),主题(弹弓/喂食)差异化清晰。

## 2. 玩法总览

**单屏正视草地**(静态层一次画完):左下角一把**弹弓**(Y 形叉 + 皮筋);画面里一只**张嘴小动物**
(位置每轮变:草坡上/树杈间/云朵边);背景树、云、篱笆(纯装饰)。

**把摇杆往后拉**:位移向量决定发射——**方向 = 与拉相反**(真弹弓)、**大小 = 力度**;拉动时画面
实时画一条**虚线抛物线预览**(拉多远弧多远,幼儿靠"让弧对准嘴"自学瞄准,绕过"反向"的反直觉)。
**松手**(摇杆弹回)→ 果子按抛物线飞出。落进/贴近动物嘴 → **吃掉**(嚼两下 + 满足)+ 喂饱度+1;
喂饱 → 动物蹦跳**长大** → 换下一只(收集)。喂够一批动物 → **全场派对**。没喂中 → 果子落地弹一下
**变成一朵小花**(零失败,不是错),动物做"还要~"表情。

> **一次一颗果子在飞**(busy_bus 容量-1 纪律,因果链单一);果子落定前弹弓装填下一颗但不可再发。
> **摇杆 Z 键 = 备用发射键**(降级模式,§3/§11①)。

## 3. 输入 → 机制映射

| 输入 | 复用的读数 | 机制 |
|---|---|---|
| 摇杆 X/Y(归一化,软归中) | `poll_joy`(死区/归中/轴向抄定案) | **拉力向量 `pull`**;`\|pull\|>AIM_MIN` 时显示抛物线预览;发射初速 `v0 = -pull × LAUNCH_POWER`(反向=真弹弓) |
| 松手弹回(边沿) | `\|pull\|` 从 >`AIM_MIN` 跌回 <`RELEASE_THRESH` | **发射**:力度=松手前短历史窗的**峰值**,角度=松手边沿前**最后稳定采样的方向**(=预览弧正显示的方向,§5.1 所见即所得铁律)。**默认发射方式**(拉-放,最贴真弹弓) |
| 摇杆 Z 键(`press_edge`) | `poll_joy` 已有 | **备用发射**(`FIRE_MODE=Z` 时:拉住瞄、按 Z 发);或彩蛋"抖弓" |
| 节点 RGB | `node_rgb`(节流) | 拉力度 → 亮度(蓄力可见);命中=绿闪;派对跑彩虹(`hue2rgb`) |

## 4. 状态机

```
[SCAN] ──(scan_bus 认领到 joystick)──► [ROUND_SETUP] ──► [AIM]
   │ 没认到 → 无字提示卡(摇杆+插头图)+ 周期重扫(通用形态)

[ROUND_SETUP] 放一只张嘴动物(从位置表随机取,不重复上一处),弹弓装填一颗果子
[AIM]         20Hz:poll_joy → 拉力向量 → \|pull\|>AIM_MIN 画抛物线预览 + 节点 RGB 蓄力亮
[AIM] ──(松手弹回 / FIRE_MODE=Z 按 Z)──► [FLIGHT]
[FLIGHT]      果子抛物线积分(§5.2),40~60fps;命中判定(§5.3)
[FLIGHT] ──(入嘴容差内)──► [EAT](嚼+喂饱度+1)   ──(落地/飞出)──► [MISS](落地变小花)
[EAT] ──(喂饱度 < FEED_PER_ANIMAL)──► [ROUND_SETUP 同一只,重装填]
[EAT] ──(喂饱)──► [GROW](蹦跳长大 ~600ms)──► [ROUND_SETUP 换下一只]
[MISS] ──► [ROUND_SETUP 同一只,重装填]

跨状态:喂够 `ANIMAL_QUOTA` 只 ──► [PARTY](~3.5s 彩虹+彩纸)──► 清批换新 ──► [ROUND_SETUP]

打盹/深度省电:core2_sleep 全托管(桌面玩法,kick=poll_joy activity)。
DEEP 唤醒:切 5V→节点复位→重扫;弹弓/果子**复位安全态**(弹弓装填待命、飞行中果子作废重填、
已喂饱进度不清——chain_lab "复位安全位置、不吞进度"原则)。
```

- **发射门控**:`FLIGHT`/`EAT`/`GROW`/`PARTY` 期间不可发射、不排队、不二次触发;但**拉动仍有视觉
  反馈——空皮筋跟手拉伸**(未装填,无预览弧、弹不出果子)。原则 2 要求任何动作 <100ms 有反馈,
  输入永远不静默。发射后另有 `FIRE_LOCKOUT_MS` 短锁(§5.1)。

## 5. 核心机制细则

### 5.1 拉-放检测(历史窗采样 + 松手边沿)

- 每帧记 `pull` 向量入**短历史窗**(~4 样本);进入"蓄力"(`|pull|>AIM_MIN`)后开始跟踪。
- 🔴 **发射参数拆两半(所见即所得铁律)**:**力度 = 历史窗内 `|pull|` 峰值**(抗回弹瞬间量值塌缩);
  **角度 = 松手边沿前最后 1~2 个稳定采样的方向**(= 预览弧正在显示的方向)。**严禁角度也取峰值时刻**
  ——幼儿常见操作是"拉满 → 保持满拉横扫瞄准 → 松手",若用峰值时刻的角度,飞出方向≠预览弧方向,
  自教机制(§11② 的唯一解法)当场崩塌。**不变式:发射弹道 ≡ 松手前一帧预览弧。**
- **松手边沿**(`FIRE_MODE=0`):`|pull|` 由 >`AIM_MIN` 跌到 <`RELEASE_THRESH` → 判定"松手发射"。
  `RELEASE_WINDOW_MS` **两态实机定案**:0 = 跌破即发(默认——任何回缩都算射出,最贴真弹弓与幼儿
  猛放,最防"我松手了却没反应");>0 = 要求 ≤ 窗口内跌落(仅当实机出现"调整拉力时误发射"再启)。
- **蓄力中止复位**:未触发发射、`|pull|` 落回 <`AIM_MIN` → 清历史窗/峰值回待机;残留峰值不得污染
  下一次发射。
- **发射后锁定 `FIRE_LOCKOUT_MS`(~250ms)**:摇杆回中过冲震荡可能反向瞬间越过 `AIM_MIN`,锁定期内
  忽略蓄力判定,防幻影预览闪烁/幻影发射(FLIGHT 门控盖大头,此锁盖 EAT→ROUND_SETUP 快速路径)。
- **降级**:`FIRE_MODE=1` 时改为拉住瞄准、按 Z 发射(松手检测彻底不可靠时的兜底;busy_bus
  `BUS_DRIVE_MODE` 式一行切换,与 window 两态合计三变体实机定案,见 §11①)。

### 5.2 抛物线(弹道积分)

- `v = v0;每帧 v.y += GRAVITY×dt;pos += v×dt`。飞行时间/出界封顶,防果子飞太久或卡屏外。
- **抛物线预览**(AIM 态):用同一套 `v0=-pull×LAUNCH_POWER` 预演几十步,采样 `TRAJ_DOTS` 个点画
  虚线(小圆点脏矩形)——拉多远弧多远,当场可见因果。**预览与发射共用同一积分函数、同一参数来源**
  (§5.1 不变式),保证"弧对准嘴 = 真能喂进嘴"。

### 5.3 命中判定(容差给够)

- 果子飞行中每帧算与**动物嘴中心**距离,< `HIT_TOL_PX` → `EAT`(容差覆盖幼儿摇杆抖动,宁大勿小)。
- 未命中且落地或飞出 → `MISS`。

### 5.4 喂饱 / 长大(影子变量纪律)

- `EAT`:嚼动画(嘴闭合两下)+ 满足短音 + **五声音阶第 n 音**(n=本只已喂数,进度可听)+
  `HAPTIC_COLLECT`;`feed++`。
- `feed==FEED_PER_ANIMAL`:`GROW`——动物缩放放大 + 蹦跳 + 欢叫;完成判定用 game_task 侧帧倒计时、
  逻辑态存影子变量(不跨任务碰 LVGL,busy_knobs 小鸟 / chick_pour 捕获先例)。

### 5.5 零失败处理

- **miss 零惩罚**:果子落地弹一下变小花(装点草地),动物"还要~"上扬表情,立刻可再拉;不扣分、
  无负面音色、带 `MISS_SND_COOLDOWN_MS` 节流。
- **动物位置每轮变但静止**(不做移动目标——那是 fish_pond 的轴);位置从手工位置表取,加载校验。
  🔴 **校验准则 = "满拉状态下仅靠角度可命中"**,用**真实可达包络(safety parabola)**判:以弹弓
  锚点为原点(y 向上),满拉初速 v=`LAUNCH_POWER` 时可达 ⇔ `y ≤ v²/(2·GRAVITY) − GRAVITY·x²/(2·v²)`
  (再留 ~10% 余量;包络内任意点都有高弧/低弧两解)。⚠️ **不可用简化圆形判据"距离 ≤ 0.9×v²/GRAVITY"**
  ——那是水平射程,对高处目标(树杈/云朵)会高估约一倍可达性(垂直极限仅 v²/2g),实算已发现误判点
  (2026-07-15 实现期核实)。幼儿必然每次拉到物理满行程,**力度维实机会塌缩成常数,真技能轴是角度**;
  需要轻拉才够得着的近位、满拉也够不到的远位都剔掉,不放"打不到/必须温柔拉"的位置。

## 7. 渲染预算合规(根 `CLAUDE.md` §6)

- **静态层**(进场画一次):草地/树/云/篱笆/弹弓 Y 叉。动物 = 预画张嘴/闭嘴 + 小/大两态切换,
  不做运行时缩放发光。
- **动态层**:同一时刻**只有一颗果子**在飞(~16×16 脏矩形 ×2 ≈ 1k px/帧,60fps 余量足);皮筋拉伸
  = 两条细线脏矩形;抛物线预览 = `TRAJ_DOTS` 个小圆点(仅 AIM 态、拉动时重算)。
- **命中/长大/miss 小花**:预渲染精灵切换 / 单精灵短脉冲,不叠加超预算。miss 小花**落定即一次性
  烘进静态层**(单次 blit,零每帧成本),数量达 `MISS_FLOWER_MAX` 或换批派对后清场重画,防对象无限累积。
- **派对彩纸**:限量 ≤8 片,15–20fps 庆祝档;全程扁平不透明色块(RGB565 banding 防线)+ init
  烘焙精灵,**零运行时 alpha、零整屏重绘**。

## 9. `tuning.h`(★ = 最可能要实机标定)

```c
// ── 拉-放弹射(本作成败核心,P1 单独验证)──────────────
#define FIRE_MODE           0     // ★ 0=松手发射(真弹弓,默认) / 1=按 Z 发射(降级)
#define AIM_MIN             0.30f // 进入蓄力/显示预览的最小拉动量(归一化)
#define RELEASE_THRESH      0.12f // 判"松手"的回中阈值
#define RELEASE_WINDOW_MS   0     // ★ 0=跌破即发(默认,最贴幼儿猛放)/ >0=窗口内跌落才算(防慢缩回误判),实机 A/B(§5.1/§11①)
#define FIRE_LOCKOUT_MS     250   // 发射后输入锁定,防回中震荡幻影蓄力(§5.1)
#define PULL_HIST_LEN       4     // 拉力短历史窗样本数(力度取峰值、角度取松手前末稳定样,§5.1)
#define LAUNCH_POWER        420   // ★ px/s per 单位拉力;满拉水平射程≈P²/G(420→~340px;320 只有 ~197px 够不到远目标),垂直极限仅 P²/2G≈170px,高处目标按 §5.5 包络算
#define GRAVITY             520   // ★ px/s^2;与 POWER 共同决定弧型
#define TRAJ_DOTS           8     // 预览虚线点数

// ── 命中 / 喂饱 / 收集 ────────────────────────────────
#define HIT_TOL_PX          28    // ★ 入嘴容差,宁大勿小(覆盖手抖)
#define FEED_PER_ANIMAL     3     // ★ 每只喂几口长大(音阶长度)
#define ANIMAL_QUOTA        4     // 喂够几只触发派对
#define MISS_SND_COOLDOWN_MS 400
#define MISS_FLOWER_MAX     6     // 草地 miss 小花上限,满了/换批清场(烘进静态层,§7)

// ── 节奏 ─────────────────────────────────────────────
#define GROW_MS             600
#define PARTY_HOLD_MS       3500
```

## 10. 电源 / 省电(桌面玩法,坑照 chain_lab/busy_bus)

- **kick 判据** = `poll_joy` 的 `activity`(拉动/按键任一),现成喂 `core2_sleep_kick`。
- **NAP**:5V 还在,拉摇杆可唤醒。
- **DEEP**:切 5V → 节点断电复位 → 唤醒重扫 + **弹弓装填待命、飞行中果子作废重填、当前动物保留、
  已喂饱进度不清**(chain_lab "复位安全位置、不吞进度"原则)。
- **拔线**:连续 ~20 帧读失败 → 提示卡;插回重扫接管。

## 11. 开放风险(实机验证时决策)

1. **"拉-放弹射"手感是本作最大未知**(P1 单独验证):松手检测**三变体一行切换、实机定案**——
   ① `RELEASE_WINDOW_MS=0` 跌破即发(默认:任何回缩=射出,最贴真弹弓与幼儿猛放,最防"我松手了
   却没反应");② `RELEASE_WINDOW_MS>0` 窗口判快缩回(仅当实机出现"调整拉力时误发射"再启);
   ③ `FIRE_MODE=1` 拉住瞄、按 Z 发(检测彻底不可靠时的兜底)。busy_bus `BUS_DRIVE_MODE` 同款纪律。
2. **"拉反向"的反直觉**:靠**实时抛物线预览**自教(孩子拉到"弧对准嘴"就松手,不必理解反向);
   验收标准同 magic_wand:无讲解 2 分钟内能靠预览弧命中。不行则把预览弧做得更醒目 / 动物嘴上加
   落点光圈呼应。
3. **抛物线可控性**:`LAUNCH_POWER`/`GRAVITY` 标定——太飘(小动作大偏移)或太钝(拉满才够到)都不行;
   动物位置表只放"可达且需一点瞄准"的点(太近无挑战、太远够不到都剔掉)。
4. **喂食匹配(动物想要哪种果)**:v1 不做(任何果子喂任何动物,果色仅装饰),避免"选果+瞄准"双决策
   超载;留作趣味增量批候选(图泡显想要的果 + Z 键循环选果)。
5. **链路强度**:孩子会用力猛拉摇杆,链路稳定性抽查沿用 chain_lab §11⑤ 结论;掉链时 `chain_bus_sniff`
   排障走 chain_lab 卡带(本作不内置诊断模式)。
6. **硬件磨损面为本作独有**:玩法本质 = 反复满力猛拉 + 弹簧全力回击,机械压力远超 chain_lab/busy_bus
   的温和偏转,软件救不了。实机专门观察节点松动与零偏漂移加快(磨损会放大软归中漂移;必要时在
   ROUND_SETUP 静止间隙重采归中,作为趣味增量批候选)。

## 12. 里程碑与验收(先验弹射手感再盖楼,busy_bus/magic_wand 教训)
