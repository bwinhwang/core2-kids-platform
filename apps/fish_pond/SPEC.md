## 0. 立项依据(为什么是它,为什么是现在这个形态)

对准 ROADMAP §0 归因四条 + §4 新约束:

1. **实体感 = 摇杆 + 曲柄**:与 chain_lab 同套可抓握输入,机械行程/阻尼天生自带本体反馈。
2. **连续比例映射 ×3**:摇杆推多少 → 船开多快;曲柄摇多少 → 线放/收多长;摇得快 → 收得快。
3. **决策随技能成长(核心新轴)**:**移动目标追踪/预判**——三强的目标全是静的,这是全平台
   第一次"目标会动"。3 岁追着鱼尾巴放饵,4 岁学会把饵埋伏到鱼的行进路径前方,成长肉眼可见;
   叠加**选鱼决策**(快条纹的浅层鱼 vs 又深又重的大懒鱼)与**收线节奏**(纯运动技能)。
4. **收集/庆祝元循环**:木桶集 3 条 → 放生彩虹派对(活的循环,非静态展示架);金鱼/鲸鱼彩蛋。
5. **大对象约束原生(本作身份)**:同屏只有船、饵、两条鱼、木桶 ≤5 个大对象;**鱼的脸就是
   游戏**——发现饵瞪眼、靠近张嘴、咬钩猛合,信息全部由大对象的**状态帧切换**承载(表情帧 =
   预烘交换,零运行时 alpha)。"少而大"同时满足护眼(§8)、帧预算(§6)、幼儿认知带宽三件事。

**与 chain_lab 的差异化**(ROADMAP 记过"同族疲劳"风险):控制拓扑相同(横向+深度)反而是
幼儿的**技能迁移优势**;新鲜感全押在**目标会动 + 表情戏剧 + 放生循环**上,见 §12-③。

## 2. 玩法总览

**单屏侧视池塘**(静态层一次画完):上沿天空带,一条**小船**(96×40)载着钓娃左右开;
水线下两层水色带(浅层/深层),**大鱼在水里悠悠巡游**;船右侧水面漂一只**木桶**(72×64,
跨水线半浸)。船上垂一根钓线,线端挂一颗**大浆果饵**(64×64)。

**摇杆左右 → 船开**(饵跟着船缓缓摆过去);**曲柄 → 放线/收线**(饵下沉/上提)。把饵放到
大鱼跟前:鱼**瞪大眼睛**转过来 → 越游越近**嘴越张越大** → **咔嗯!咬钩**(声+震+灯齐发)→
摇曲柄往上收,鱼摆尾扑腾(震动连发,拉扯感)→ **噗通出水** → 弧线落进木桶(桶鼓一下)。
**集满 3 条 → 放生派对**:三条鱼画着彩虹弧跳回池塘,全屏庆祝,新回合开始。

```
0   ┌────────────────────────────────┐
    │ ☀        🚤(96×40)     ┌木桶┐  │  天空带
48  ├~~~~~~~~~~~~│~~~~~~~~~~~│72×64│~~┤  水线(桶半浸漂着)
    │            │钓线        └────┘  │
    │          ◉饵(64×64)             │  浅层:胖胖鱼 96×72
144 ├ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─ ─┤
    │        ●●●大懒鱼(128×96)        │  深层:更重,收线更费劲
240 └────────────────────────────────┘
```

> **零失败**:没咬到 → 鱼咯咯吐泡泡转个圈回来,饵永远可以再放;收线中途停手 → 鱼在线上
> 打盹(zzz 泡),**永不脱钩**;无计时、无扣分。永远只有"还没咬"和"咬了"。

## 6. 核心机制细则

### 6.1 船与饵(宽容物理)

- 船:`vx = BOAT_SPEED_MAX × 摇杆偏移比例`(死区内 0),x 夹在 `[BOAT_MIN_X, BOAT_MAX_X]`
  (右侧给木桶让位);撞边缘滑停不粘。
- 饵:目标位 =(船 x,水线 + 线长);**双轴缓动 τ≈150ms**(§2 宽容物理:宁慢勿飘),横移时
  带轻微钟摆感(缓动天然产生,不做真摆模拟)。
- 线:`lv_line` 2px,船艄 → 饵顶,每帧更新(便宜)。

### 6.2 鱼 AI(状态帧 = 表情戏剧)

```
PATROL(巡游) → NOTICE(瞪眼) → APPROACH(张嘴凑近) → CHOMP(咬合=上钩)
     ▲                                    │ 饵被拉出感知圈
     └────── GIGGLE(咯咯吐泡泡,~0.8s)◄────┘
```

- **PATROL**:水平往返(层内),`FISH_*_SPEED`,到 x 边缘掉头(换向 = 换镜像帧);
  游动 A/B 两帧摆尾(~2fps 切换,氛围档)。
- **NOTICE**:饵进 `SENSE_R`(80px)→ 瞪眼帧 + "哦?"两音 + 灯带向鱼色渐亮;转向饵,
  以 `APPROACH_MULT × 巡速` 靠近。
- **APPROACH**:距离 < `MOUTH_R`(40px)→ 张嘴帧(嘴随距离越张越大 = 2 级预烘帧)。
- **CHOMP**:嘴锚点与饵中心距 < `BITE_R`(30px,**宽容判定,嘴附近就算**)→ 咬合帧 +
  咔嗯! + `HAPTIC_BUMP_HARD` + 灯带闪鱼色 → 进 HOOKED。
- **GIGGLE**(零失败反馈):饵离开感知圈 → 咯咯泡泡 2 帧 + 轻快音(**中性,非失败音色**)
  → 回 PATROL。有 `GIGGLE_COOLDOWN_MS` 节流防连刷。

### 6.3 收线(HOOKED,拉扯感全靠声+震)

- 鱼锚定在饵上(合并为一个精灵组),曲柄反转 → 线长减 `CRANK_PX_PER_DET`/格;
  **大懒鱼 `REEL_LAZY_RATIO=0.5`**(同样摇一格只升一半——"这条好重!")。
- 拉扯戏:鱼横向 `WIGGLE_AMP=8px` @2Hz 摆 + **触觉 tick 连发,密度∝摇速**(曲柄无力反馈,
  拉扯感的唯一触觉来源)+ 棘轮声渐密。
- **停手 → 鱼打盹**:zzz 泡 + 摆动停,永不脱钩;再摇继续。**不存在"跑鱼"**。
- 出水(线长 < `LINE_MIN_PX`):SPLASH 3 帧 + 上扬滑音 + `HAPTIC_COLLECT` → 罐头动画
  ~0.8s 鱼弧线入桶(单精灵抛物线,小脏矩形)→ 桶鼓胀帧 + 噗通。

### 6.4 木桶与放生派对

- 桶 3 态帧:空 / 有鱼(探出个鱼尾)/ 鼓胀(大鱼塞进去桶变形,喜剧感);桶上 1~3 个大圆点
  计数(不识字进度,chick_pour 音阶同理)。
- 满 `BUCKET_FULL_N=3` → **RELEASE_PARTY**:三条鱼依次画彩虹弧跳回池塘 + `SND_WIN` +
  `HAPTIC_WIN` + `LED_FX_WIN` 彩虹,~3.5s(全屏级动效,庆祝豁免 §6.2;柔和飘落,无频闪 §8)
  → 新回合。**放生**(而非囤积)既是与 chain_lab 展示架的差异化,也天然循环无终局。

### 6.5 帧预算核算(根 `CLAUDE.md` §6.2,预算 ≤15k px/帧)

| 项 | 面积 | 更新率 | 摊到 60fps 每帧 |
|---|---|---|---|
| 饵(+线) | 4,096 + ~400 | 60fps(仅输入时) | ~4.5k |
| 胖胖鱼 | 6,912 | 20fps,**与懒鱼错帧** | ~2.3k |
| 大懒鱼 | 12,288 | 20fps | ~4.1k |
| 泡泡氛围 ≤4 颗 | ~144×4 | 5–10fps | ~0.1k |
| **常态合计** | | | **~11k ✅** |

- 收线时:鱼+饵合并脏矩形 ~128×110 @30fps ≈ 7k/帧 ✅(此时鱼不再单独巡游)。
- 两鱼错帧更新是硬要求(同帧齐动峰值 ~23k 超预算)。
- 静态层(天空/双层水色带/水草石头/桶底)进关烘一次进 PSRAM canvas(150KB);
  精灵表两鱼种 ×6 态 ×左右向 + 船/饵/桶/水花 ≈ 0.5MB PSRAM,直映射 4MB 内绰绰有余。
- **全程扁平不透明色块(RGB565 banding 防线),表情/嘴型全部预烘帧交换,零运行时 alpha,
  零整屏重绘**(唯一整屏级 = RELEASE_PARTY,庆祝豁免)。

### 6.6 精灵帧清单(init 预烘)

- 每鱼种 6 态 × 左右镜像 = 12 帧:游 A / 游 B / 瞪眼 / 张嘴(小/大 2 级)/ 咬合 / 上钩摆;
- 船 2 帧(闲/摇臂),饵 1,桶 3 态,水花 3 帧,咯咯泡泡 2 帧,zzz 泡 1,入水波纹 2 帧。

## 10. 电源 / 省电(桌面玩法,坑照 chain_lab)

  ——否则钓着钓着打盹;每帧照常 `core2_sleep_feed()` 喂 IMU。
