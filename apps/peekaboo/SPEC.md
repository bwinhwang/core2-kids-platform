## 0. 为什么重写(诊断)

v1 的问题不在完成度,在**玩法只有 1 bit 的信息量**:

1. **只有一个动词**:捂 → 松。第 1 次和第 100 次的结果完全相同——同一只圆圆、同一个伸懒腰、同一声上扬音。
2. **DLight 给的是连续照度,游戏只用了黑/亮 1 bit**,输入丰富度被扔掉。
3. **没有悬念**:现实躲猫猫的核心是"手后面会是什么"的期待与揭晓惊喜;v1 揭晓的永远是已知答案。
4. **没有积累**:5 个点 → 庆祝 → 清零。3~4 岁最吃"重复但有变化"(again! but different),v1 只有重复没有变化。

## 1. 核心概念:「夜里来客」

把昼夜循环从"开关灯"改造成**两幕小戏剧**:

- **夜 = 悬念**:捂住入夜后,黑暗里有动静——草丛晃、**一对发光的眼睛眨**、一声**闷闷的叫声**。"有谁来了……是谁呀?"
- **天亮 = 揭晓**:松开手,**每次从访客池出场一位不同的小客人**(专属造型/叫声/出场动画),和圆圆打招呼。
- **积累 = 相册**:揭晓过的访客头像点亮;集齐 6 位 → **全员大游行**庆祝,开新一轮。
- **变化 = 玩法回应玩法**:猛地松开 → 客人吓一跳;慢慢掀缝 → 客人也偷偷探头;夜捂得久 → 萤火虫攒多、圆圆做梦,天亮时梦里的蝴蝶飞出来;小概率**彩虹鸟**稀客空降(jackpot)。

零失败不变:没有"猜错",不管怎么松开都是惊喜,只是惊喜的"味道"不同。全程无文字:悬念靠眼睛+叫声,揭晓靠形象+音签,收集靠头像。

## 3. 状态机(游戏层)

```
                 ┌────────────(游行结束:相册清灰、池重置、按当前照度重评昼夜)─────────────┐
                 ▼                                                                    │
 [DAY_IDLE] ──捂(lux<ref×COVER)──► [NIGHT_TEASE] ──松(lux>ref×UNCOVER)──► [DAWN_REVEAL] ─┤
   白天待机:访客陪圆圆              夜悬念时间线(§6)          出场编排 ~0.5–1.2s(§7)      │
                 ▲                                              │(第 6 位入册)          │
                 └──────────────────────────────────────────────┴──► [PARADE ~4.5s] ────┘
```

- **DUSK 黄昏**(P4,`DUSK_ENABLED`):照度在迟滞带内驻留形成的**纯装饰第三场景**(§9),不参与访客逻辑。
- **省电三态**(core2_sleep)与此正交:非 AWAKE 冻结昼夜转换(v1 既有)+ 冻结悬念/梦泡/萤火虫计时;
  唤醒回 AWAKE 时悬念节拍**重置**(眼睛若已出现则保持)。
- DAWN_REVEAL 是短暂的出场编排期,不阻塞照度判定(打断规则见 §15)。

## 4. 传感层扩展:照度 → 三条信息

`on_lux()` 的基准/迟滞逻辑不动,在其上**多提取两条信息**,连同状态跳变共三条供游戏层:

1. **状态跳变**(既有):入夜 / 天亮。
2. **松开速度档**(P3):状态为夜时,统计照度处于迟滞带 `(ref×COVER, ref×UNCOVER)` 内的**连续读数次数**
   `s_band_reads`(一次读 ≈132ms;跌回捂住区则清零)。天亮瞬间分类:
   `≤DAWN_FAST_READS(2)` = **猛松**;`≥DAWN_SLOW_READS(6)` = **慢掀**;其余 = 普通。
3. **夜时长**(P2):AWAKE 且为夜的帧计数 `s_night_frames`;天亮时 `≥NIGHT_LONG_S(6s)` = **长夜**(有梦)。

P4 黄昏判据也在此层:迟滞带内连续 `DUSK_DWELL_READS(3)` 次读数(~400ms)→ 进黄昏;越过任一阈值即离开。

### 5.2 抽签规则(入夜时定,不是天亮时)

- 入夜瞬间从**本轮未收集**的访客里均匀随机抽一位(悬念内容要匹配来客,故必须夜里就定);
  新一轮首抽**不与上一轮最后一位重复**(防"刚游行完又是它")。
- 抽签后掷稀客骰(P3):`RARE_PCT(8%)` 命中 → 本夜改为彩虹鸟,原抽签作废退回池。
  彩虹鸟**不入册、不占轮次**,是纯 jackpot。

## 6. 夜悬念时间线(P2)

入夜(t=0)先走既有编排:场景切换 + 下行风铃(v1 音)+ `HAPTIC_BUMP_LIGHT` + 灯带转暗呼吸;
**白天在场的访客瞬时隐藏**("回去躲了",整屏切换掩护,无需动画)。然后:

| 时刻 | 事件 | 实现 |
|---|---|---|
| ~0.8s (`TEASE_RUSTLE_MS`) | **草丛有动静** | 该访客藏点草丛 x ±2px 晃 2~3 下 + 窸窣 3 短音 {1200,25,18}{0,40,0}{1400,25,15}(极轻) |
| ~1.5s (`TEASE_EYES_MS`) | **眼睛亮起** | 按访客夜眼样式(表 §5.1)在藏点显示,低频眨(opa 90↔255 二值切换) |
| ~2.5s 起,每 ~3s (`TEASE_CALL_PERIOD_MS`) | **闷闷的叫声** | 音签闷化:freq×75%(下限 200Hz,小喇叭失真)、ms×1.2(慢一点)、amp×40% |
| 整夜持续 | **等待有产出** | 梦泡泡:圆圆头顶 3 档离散长大(10/16/22px 淡蓝白 `0xDDE6F5`,每 `DREAM_STEP_S`=2s 一档);萤火虫从 2 只每 2s +1 至 6 只;星星照旧眨 |

- **彩虹鸟之夜**:无动静、无眼睛、无叫声——异常安静就是它的悬念;改 ~1.2s 一颗**流星**划过夜空
  (12×4 斜条一次性 600ms 位移后隐藏)+ 极轻一声 {1568,50,20}。
- 孩子提前松手 → 跳过剩余节拍直接揭晓(零失败/即时因果);眼睛熄灭与场景切换同帧。
- 悬念计时只在 AWAKE 走;打盹中被照度变化唤醒后节拍重置(眼睛已出现则保持)。

## 7. 揭晓编排(天亮)

天亮瞬间(整屏切换时刻,v1 既有)基础包:日场景 + 圆圆睁眼弹跳 + `HAPTIC_WAKE` + `LED_FX_COLLECT`;
**主音改为访客音签全响**(v1 的 `SND_HELLO` 只保留给开机/单元接管)。随后:

- **出场动画**按 §5.1 表(0.5~0.9s),从藏点草丛位置进场到站位;圆圆瞳孔**看向访客一侧**
  (±`GAZE_DX`=3px,保持 `GAZE_MS`=2s 后回中,busy_knobs 同款手法)。
- **首遇入册**:出场收尾时相册头像点亮 + `SND_COLLECT` + `HAPTIC_COLLECT`(错开音签,靠音频队列串行)。
- **白天待机**:访客留场陪圆圆,y ±2px 低频 bob(1.2s 循环);下次入夜隐藏。

**变体(P3,按 §4 速度档)**:

| 档 | 视觉 | 音频 | 触觉 | 灯带 |
|---|---|---|---|---|
| **猛松** | 出场时长×0.6、弹跳高度×1.5,圆圆也吓一跳(bounce 加大) | 前置"哇"两音 {1046,50,65}{784,90,55},再接音签 | `HAPTIC_BUMP_MED` | `LED_FX_FLASH` |
| **慢掀** | 两段出场:先探半个头停 `ENTR_PEEK_HOLD_MS`=400ms,再完全出来(总×1.5) | 音签 amp×0.7(轻柔) | `HAPTIC_HELLO` | `LED_FX_GATHER` |

**长夜有梦(P2)**:夜时长 ≥`NIGHT_LONG_S`=6s → 天亮时梦泡泡"啵"消失,2~3 只小蝴蝶

停留本个白天,下次入夜飞走(隐藏),不入册。

## 8. 相册与游行

- **相册**:顶部一排 6 枚 18×18 头像位(x=8 起、间距 4、y=6),替代 v1 计数 dots。
  每枚 = 圆底 + 1~2 特征块(猫耳/鹰眼圈/蛙泡眼/兔耳/鸭喙/猬刺),≤3 对象;
  未集 = 灰(底 `0xCFC6B8` 特征 `0xB8B0A2`),已集 = 访客主色。
- **游行**(第 6 位入册后 `PARADE_LEAD_IN_MS`=600ms):6 位访客从左屏外横穿到右屏外
  (各 `PARADE_WALK_MS`=3s,逐个错峰 `PARADE_STAGGER_MS`=250ms,y ±3 bob,走中景 y≈160)
  + 圆圆原地欢跳三次 + 彩纸(复用 burst)+ `SND_WIN` + `HAPTIC_WIN` + `LED_FX_WIN` 彩虹。
- **游行收场**:相册清灰、访客池重置、(P4)游行数 +1 并写 NVS;按当前照度带重评昼夜(§15 规则 1)。
- (P4)相册右侧显示游行小星 ✦ × `min(count, PARADE_STARS_MAX=5)`(14px)。

## 9. 黄昏态(P4,`DUSK_ENABLED`,实机不稳可一键关)

- **判据**:§4 的迟滞带驻留(~400ms),双向都可进(捂到一半 / 掀到一半);越过任一阈值即走正常入夜/天亮流程。
- **场景**:第三张静态场景——金橙天空 `0xE8A15C`、低垂半轮太阳、暖棕草地;圆圆眯眼(睡眼线);
  单音轻风铃 {494,140,35};无震;灯带维持 AMBIENT。
- **纯装饰**:不影响抽签/相册/悬念(黄昏落夜后悬念时间线正常从 t=0 起)。
  与慢掀天然联动:慢慢掀 = 先路过金色晨曦再天亮,是加分体验。
- 切换也是整屏级、事件驱动,渲染红线不破。

## 12. 渲染预算合规(根 `CLAUDE.md` §6)

- **整屏重绘时刻**:入夜/天亮/黄昏切换、游行开场——全部事件级,永不每帧。
- **夜常态动像素**:星 6×~8px + 萤火虫 ≤6×9px + 眼睛 2×~10px + 梦泡泡低频换档 + 草丛偶发
  ≈ 合计 <2,500px/帧,远在 15,000px 预算内。
- **白天常态**:访客 idle bob(~44×40 低频)+ 圆圆眨眼 ≈ <3,500px/帧。
- **出场**:单组 ~1,760px 移动 0.5~0.9s,与 tilt_maze 球同量级。
- **游行**:庆祝档(§6.5 允许 15~20fps);6 组错峰 + burst 8 颗,**不再加其它动效**。
- 全部不透明色块,零逐帧 alpha;预建 HIDDEN 组不参与渲染(实现前 MCP 确认 LVGL 版本行为)。

## 15. 交互优先级 / 冲突矩阵(结算顺序)

1. **游行最高**:期间照度转换**挂起**(只记当前带,不切场景);游行结束按当前照度重评昼夜
   (捂着就直接入夜走悬念)。悬念计时游行中不走。
2. **出场中又入夜**:`lv_anim_delete` 配对掐出场动画 → 访客瞬移站位 → 随场景整体隐藏 → 正常抽签走夜。
3. **快速捂/松连打**:每次天亮都出访客(零失败/即时因果),悬念节拍被跳过是预期行为;
   连打 6 次集齐照样游行,游行挂起转换天然降速。不设冷却、不惩罚。
4. **省电**:非 AWAKE 冻结昼夜转换(v1)+ 冻结一切装饰计时;装饰活动不 kick。
   NAP 中捂/松照度变化仍可唤醒(v1);DEEP 断 5V → 唤醒后 `unit_attach` 重接管(v1)。
5. **拔线**:任意态 → `unit_lost` 提示卡(v1);访客/眼睛就地冻结,重插继续。
6. **稀客**不打断轮次、不入册;黄昏(P4)纯装饰,永不影响 1~5。
