## 玩法 = 二值昼夜(捂/松)+ 揭晓计数

DLight 测环境光,小手把传感器**一捂(挡光)→ 入夜**(深蓝夜空 + 月亮 + 星星眨 + 萤火虫漂 + 圆圆闭眼睡 + "Zzz")、**一松(光回)→ 天亮**(蓝天 + 太阳 + 圆圆睁眼伸懒腰 = 躲猫猫揭晓)。每次天亮 = 一次揭晓计数,攒够 `WIN_CYCLES=5` → 全屏小庆祝(星星迸发 + 过关音 + 三连震 + 彩虹)。零失败、无计时。

四通道:切换瞬间 换场景 + 音(天亮=SND_HELLO 上扬/入夜=下行风铃 play_notes)+ 轻震(HAPTIC_WAKE / BUMP_LIGHT)+ 灯带(白天 AMBIENT 暖亮 / 夜晚 IDLE 暗呼吸)。

## 🔴 昼夜是离散二值状态,不是每帧连续调色

正因如此才守得住 root `CLAUDE.md` §6 渲染红线:整屏级重绘只在"捂/松"那一刻(像 tilt_maze 进关),日常帧只有限量小精灵(6 星眨 + 4 萤火虫漂,lv_anim 局部脏矩形)在动。**别把它做成每帧全屏渐变**(经典 ESP32 无 2D 加速,每帧整屏 ~31ms 必崩帧率)。

## 平台新增(可复用)

`components/units/unit_dlight` —— BH1750FVI 最小驱动(init 发上电 0x01 + 选连续高分辨率 0x10;之后纯 receive 读 2 字节 16-bit 大端,lx=raw/1.2)。🔴 **连续模式下读不必先写命令 → 天然不用 repeated-start**(顺带避开 8Encoder 那类 MCU 从机组合读坑,虽 BH1750 是纯传感器不吃这套)。init 可重复调用:每次都重发上电+选模式,故深度省电切 5V 令单元断电复位后,重调 init 即自动重配连续模式。

> **Confirmed via esp-component-registry(espressif/bh1750 v2.0.0)**:连续模式发一次命令后重复读;opcode/换算按 BH1750 datasheet。

## 自适应亮度基准(关键设计)

不按房间标定绝对 lux,而是 `s_ref` 追踪"没捂住时的房间亮度"(向上跟得快 REF_RISE=0.20、向下跟得极慢 REF_FALL=0.006),**捂住 = lux 掉到 ref×0.35 以下、松开 = 升到 ref×0.60 以上**(迟滞带防抖)。这样明亮/昏暗房间都好用,无需换房间重调。

## 🔴 桌面/手持玩法省电坑

机身不动 ≠ 没人玩 —— **照度在变(手在捂/松)= 有人玩 → core2_sleep_kick**(否则玩着会打盹);打盹中捂/松可唤醒(NAP 时 5V 还在)。深度省电切 5V → DLight 断电复位,唤醒后 `unit_attach` 重接管(拿起机身才醒)。容错:没插单元 = 无字提示卡(光单元+插头图)+ 2s 重试,插上即"你好";连续 `ERR_STREAK_LOST=20` 次读失败判拔线回提示卡。

## 待实机标定(tuning.h,主要是房间光照灵敏度)

`COVER_FRAC=0.35`/`UNCOVER_FRAC=0.60`(捂/松阈值:太钝调大 COVER、误触调小)、`REF_MIN_LUX=15`(很暗房间兜底)、`LUX_ALPHA=0.40`(去抖)、`MOVE_FRAC=0.15`(唤醒灵敏度)、`DLIGHT_READ_TICKS=4`(读节奏 ≥120ms 测量周期)。
