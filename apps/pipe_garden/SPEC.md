## 0. 立项依据(为什么是它)

对准 ROADMAP §0 归因四条,并给旋钮开一条新技能轴:

1. **实体感 = 8 实体旋钮的真实阻尼**:与最受欢迎的 busy_knobs 同输入,拧的手感是触觉通道
   天生自带的本体反馈(可抓握,非对空气)。
2. **连续比例映射**:拧多少 → 管段转多少角(吸附到 4 个正交朝向),输入量与画面量成比例。
3. **决策随技能成长(核心新轴)**:**旋钮相互依赖**——要规划"哪几段转成什么朝向才接通一条
   水路",一段没对就断流;越玩越会看路。这是 busy_knobs(8 路各自独立、拧哪个哪个动、互不
   相干)根本给不了的**空间规划/连接**决策。
4. **收集/庆祝元循环**:水流过点亮沿途花,每朵开 = 五声音阶上行一音;整条接通 → 全开 +
   蝴蝶飞来 + 派对。

**新图式(接水管/水路)+ 新技能轴(连接规划)**,主题(花园/流水)全平台首次,无审美疲劳。

## 2. 玩法总览

**单屏俯视小花园**(静态层一次画完):左边缘 = **水龙头**;中间一条**水渠**由 `SEG_COUNT` 段管子
横向接成(占**最左 `SEG_COUNT` 个旋钮**,左→右顺序与旋钮台物理顺序一致——busy_knobs 空间对应
纪律);右端一排**花盆**(干渴,灰扑扑);水渠沿途有**石头/高台**,水路要转弯绕上去再下来。

拧**旋钮 i** 转**第 i 段**管子(连续跟手,松手吸附到最近正交朝向)。当某段的开口与左右邻居对齐、
且从水龙头一路连通到该段 → 水实时流到那里;**断口处水轻轻滴一下(dribble)**,一眼看出"哪里还没
接上"。整条接通 → 水哗地流到底,沿途花**逐朵绽放**(第 n 朵 = 音阶第 n 音)+ 右端大花开 + 蝴蝶飞来。
**旋钮板载 RGB**:该段已接上邻居 = 绿,否则暗(物理旋钮上就能看哪断了)。**拨动开关 = 水龙头开/关**
(默认开)。一关接通 → 蝴蝶飞来 → 换新布局(石头位置/弯道变)→ 循环。

> **零失败**:水不会漏光、无计时、无扣分;没接通只是水在断口等着,继续拧就通。永远只有"还没接上"
> 和"接上了"。

## 3. 输入 → 机制映射

| 输入 | 来源 | 机制 |
|---|---|---|
| 旋钮 i 增量 | `read_increments()`(读后自清零) | 第 i 段朝向累加旋转,松手吸附最近 90°(连续比例) |
| 拨动开关 | `read_switch()`(基线去抖) | 水龙头开/关(默认开;关掉水停,纯"我先看看") |
| 旋钮板载 RGB | `set_led(i,...)`(值不变不重发,节流) | 第 i 段连接态:接上邻居=绿,否则暗(实时断点提示) |
| 旋钮径向按键(可选) | `read_buttons()` | v1 保留不用;开放风险 §11④ 备:按=该段回正(undo 助手) |

## 4. 状态机

```
[SCAN] ──(8encoder init OK)──► [LEVEL_SETUP] ──► [PLAY]
   │ 没插/读失败 → 无字提示卡(旋钮台+插头图)+ 周期重试(通用形态)

[LEVEL_SETUP] 载入一关手工布局(水龙头/花盆/石头/各段初始乱朝向),跑可解性校验(§5.5),
              重写全部旋钮 LED 为初始连接态
[PLAY]        30Hz:read_increments → 各段朝向吸附 → 连通计算(§5.2)→ 水流可视 + 断口 dribble
              → LED/花开/反馈;read_switch 控水龙头
[PLAY] ──(从水龙头连通到最后一朵花)──► [BLOOM](逐花绽放 + 音阶,§5.4)
[BLOOM] ──► [GARDEN_PARTY](~3.5s:全花+蝴蝶群+彩纸+SND_WIN+灯带彩虹)──► [LEVEL_SETUP]

打盹/深度省电:core2_sleep 全托管(桌面玩法,kick=拧/按/拨任一 activity)。
DEEP 唤醒:切 5V→单元复位→重扫 + **重写全部 LED + 重采开关基线**(防幻影);
关卡进度(各段朝向)保留在应用侧影子变量,不随单元清零而丢。
```

## 5. 核心机制细则

### 5.1 段模型(直管 / 弯头,4 朝向)

- 每段是 **直管**或**弯头**(幼儿简化,不用 T/十字);各有 4 个 90° 朝向。开口端口用 4-bit 掩码
  表示(左/右/上/下)。旋钮增量累加 → 目标角,**松手吸附最近 90°**(`SNAP_TOL` 内直接对齐,
  仿 chain_lab 吸附;不要求精确停位)。
- 段与段之间可在**两行高度**(地面行 / 高台行)之间走,弯头负责"上台阶/下台阶"绕石头。

### 5.2 连通计算(每帧,从水龙头链式推进)

- 从水龙头出发,沿 i=0→SEG_COUNT-1 逐段检查:**段 i 的右端口开** 且 **段 i+1 的左端口开**
  且 **两者高度对齐** → 连通延伸到 i+1,否则在此断开。O(N) 便宜。
- `reach = 最远连通到的段/花索引`。这是"手工编排 + 加载校验"纪律(根 §11)针对可解性的落点:
  **布局由设计保证有解**,LEVEL_SETUP 跑一遍断言(存在一组朝向使全连通),失败退预置关。

### 5.3 水流可视 + 断口反馈

- 水实时流到 `reach` 处:连通段显示"流动高光"(沿固定路径跑的小亮点,脏矩形);断口处**滴一滴
  水 + 轻"嘀嗒"**(节流),明确指出"这里还没接上"——**建设性反馈,不是失败**。
- 拧动某段使 `reach` 前进一段 → 该段"接上"叮一声 + 旋钮 LED 转绿 + 灯带微扫。

### 5.4 花开(逐朵 + 音阶,防跨任务踩状态)

- 水流抵达某花盆 → 该花绽放动画(含苞→绽放精灵切换,~300ms)+ **五声音阶第 n 音**(n=已开花数,
  进度天然可听,chick_pour 先例)+ `HAPTIC_COLLECT`。完成判定用 game_task 侧帧倒计时、逻辑态存
  影子变量(不跨任务碰 LVGL,busy_knobs 小鸟先例)。

### 5.5 布局手工编排 + 加载校验

- 每关手工编排(水龙头/花盆/石头位置、各段类型、初始乱朝向),换关只换布局不改机制。加载跑
  可解性 + 越界断言,失败退预置关(永不失败,对应 verify 纪律)。**难度带**:P1 全直管(只需拧平)
  → 加 1 个石头(1 处绕行)→ 多石头/多花,循序渐进。

## 6. 反馈矩阵(六类事件 × 五通道,旋钮板载 RGB 为第五通道)

| 事件 | 屏幕 | 音频 | 触觉 | 底座灯带 | 旋钮板载 RGB |
|---|---|---|---|---|---|
| 拧管旋转 | 该段脏矩形转动 | 极轻"咔"(拧的 tick,~30ms;旋钮 detent 自带触觉) | —(靠旋钮机械行程) | — | — |
| 吸附对齐 | 段吸附到正交朝向 | 轻"嗒" | — | — | — |
| 接上一段(reach 前进) | 该段流动高光延伸 | 叮(上行) | `HAPTIC_BUMP_LIGHT` | 暖色微扫 | **该段转绿** |
| 断口 dribble | 断点滴水 | 轻"嘀嗒"(节流,中性) | — | — | 断点段保持暗 |

| 全接通派对 | 全花+蝴蝶群+限量彩纸 | `SND_WIN`+上行琶音 | `HAPTIC_WIN` | `LED_FX_WIN` 彩虹 | 全绿呼吸 |

## 7. 渲染预算合规(根 `CLAUDE.md` §6)

- **静态层**(进场画一次):花园地面、水龙头、花盆排、石头/高台、水渠底座框。结构再丰富几乎免费。
- **动态层**:**只有正在转的那一段管子重画**(单段小脏矩形,同一时刻幼儿只拧一个旋钮);其余段不动。
- **水流高光**:沿**已定路径**跑的小亮点(脏矩形),非整条逐帧重画;断口 dribble = 单点短脉冲。
- **花开 / 蝴蝶**:预渲染两态切换 + 蝴蝶 5–10fps 氛围档(小区域);派对彩纸限量 ≤8 片,15–20fps。
- **全程扁平不透明色块(RGB565 banding 防线)+ init 烘焙精灵,零运行时 alpha、零整屏重绘。**
  本作甚至没有"持续移动的主角"(只在流水/花开瞬间有小动画),是全平台渲染最省的一档。

## 10. 电源 / 省电(桌面玩法,坑照 busy_knobs)

- **kick 判据** = link 层 activity(任一旋钮转/按/拨动开关),喂 `core2_sleep_kick`——否则拧着拧着打盹。
- **NAP**:5V 还在,拧旋钮可唤醒。
- **DEEP**:切 5V → 单元断电复位(LED 灭、计数清零、开关基线丢)→ 唤醒重扫 + **重写全部 LED
  为当前连接态 + 重采开关基线防幻影翻转**(8Encoder README 明确);各段朝向进度存应用侧影子变量,
  不随单元清零而丢。
- **拔线**:连续 ~20 帧读失败 → 提示卡;插回重试接管。

## 11. 开放风险(实机验证时决策)

1. **"转管子接水路"的概念可达性是全场最高风险**(P1 单独验证):3~4 岁能否无讲解 2 分钟自发理解
   "拧到接上就流水"(magic_wand 成功标准)。**降级预案**:`SEG_COUNT` 降到 2~3 + `ROCK_COUNT=0`
   全直管(退成"把管子拧平"的极简版,近 busy_knobs 难度),仍成立(流水+花开+派对在);再不行回设计层。
2. **旋钮 i ↔ 段 i 的映射清晰度**:线性水渠已把段左→右对齐物理旋钮左→右(最强对应);**刻意不上
   2D 网格**(映射会立刻变难);实机若仍困惑,靠"拧哪段哪段的花园位置闪一下"强化对应。
3. **石头绕行(两行高度)可能偏难**:P1 先纯水平水渠(无高度维);绕行作为难度带的后段,实机确认
   幼儿吃得消再加。
4. **旋钮按键 undo 助手**:v1 不做;若实机见孩子把某段拧乱又找不回,加"按旋钮=该段回正"。
5. **"就差最后一段"的挫败**:靠断口 dribble + 旋钮 LED 绿点提示"哪里还没接"化解;实机专门试
   "只剩一段没对"的场景是否够引导。

## 12. 里程碑与验收(先验概念可达性再盖楼,magic_wand 教训)

**P1 概念可达性验证(先烧这个)**:花园静态层 + 最小 2~3 段**全直管**水渠 + 拧管吸附 + 连通计算 +
水流可视/断口 dribble + 接通后 1~3 朵花开。**验收:3~4 岁无讲解能否自发理解"拧到接上就流水",
拧通时"哗~花开"让孩子想再来一关。** 不过关先降 `SEG_COUNT`/`ROCK_COUNT`,再不过关回设计层,不进 P2。

**P2 规划决策 + 收集闭环**:加石头绕行(弯头 + 两行高度)+ 多花音阶 + 断口引导 + 蝴蝶派对 +
关卡库/换关 + 旋钮 LED 连接态。**验收:接通需要"看路选朝向",分错朝向只有中性 dribble 无负面;
逐花音阶可感知。**

**P3 打磨**:花/蝴蝶美术、难度带调参、launcher 专属图标(水管+花,要重刷 launcher)、`screenshot`
截图自查花园/水路/派对布局(§10.1)、flash_one.sh/flash_map.md 登记(立项确认槽位后)、README
as-built、根 CLAUDE.md §1/§12 索引更新。
