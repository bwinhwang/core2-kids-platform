## main/ 结构

```
main/  app_main.c(core2_board_init 一键 bring-up + 起状态机)
       game_state.c(60Hz 任务 + 状态机 ATTRACT/PLAY/WIN;省电托管给 core2_sleep)
       physics.c · maze.c(关卡 + BFS 校验 + 滑行碰撞)· render.c(三层 + 特效)
       feedback.c(四通道编排器)· parent_menu.c · tuning.h
```
- **任务/队列**:game_task(60Hz)读 IMU→物理→碰撞→渲染→发事件;feedback/audio/haptics/ledstrip 各自事件驱动消费队列,**不阻塞 game_task**(符合 §3.2)。
- **免校准**:无零点/校准概念,`physics_set_position` 只复位 pos/vel;低通滤波首帧样本自预热。
- **打盹/唤醒**:游戏侧每帧 `core2_sleep_feed()` 喂加速度即可;机制/踩坑见根 `CLAUDE.md` §7(电源/休眠/背光)。

## 绝对零点,免校准(2026-07-02 设计定案,推翻 §8.2)

**定案**:删除整个校准机制。"平"= 屏幕平面与地面平(重力绝对参考),规则永远一致。

**设计依据(用户拍板)**:游戏目的就是让孩子学会控制设备——把设备端平正是要教的东西,物理滚珠迷宫玩具就是这个范式(它们从不"校准"到握姿)。原 §8.2 姿态校准替孩子做了他能自己学会的动作,代价是:①同一物理倾斜在不同会话里作用不同(违背"因果一致"第一性原则);②引入隐藏的、每次都变的参考系;③滋生零点污染 bug。斜着拿球就往下滚 → 孩子自己调整设备对抗,这就是游戏本身。

**技术前提(已实机验证)**:板子加速度计偏置须小于死区 0.06g。本机平放实测 `x=-0.023 y=-0.007 z=1.065,漂移 0.001g` —— x/y 偏置不到死区一半,余量充足;z 偏差不影响玩法(只用 x/y)。

**落地变更**:
- 删 `ST_CALIBRATE` 态与 `calibrate_tick`、`physics_calibrate()`、`physics_t.zero`;ATTRACT 倾斜触发**直接进 PLAY**;
- 删家长菜单"Recal"按钮与 `game_state_request_recalibrate()`(用户明确不要修正入口);
- 删 tuning:`CALIB_*`、`IDLE_STILL_THRESH`;
- 低通滤波改为首帧样本自预热;
- **保留**:减除式死区(见下)、速度封顶、轴映射。
- **已否决**:静止自回零(偷偷移动"平"违背一致性原则)、NVS/菜单偏置修正(本机余量充足)。

**状态**:✅ 已实机验证(2026-07-02 烧录确认)。手感若偏慢,调 `TILT_GAIN` 1800→~2000,勿动死区;斜着拿球向下滚属预期。

### 根因记录:IMU"平放单向慢爬"(2026-07-02,已随免校准修复)

**症状**:有时平放机身,球仍以 ~20px/s 匀速朝固定方向爬。**根因不是 IMU 硬件**:
1. **校准零点被"开始手势"污染(主因,免校准后已消失)**:旧 `calibrate_tick` 从进状态第 1 帧无条件采样,离开 ATTRACT 的倾斜手势把零点拖偏 ≈0.06g(恰好一个死区)。
2. **硬死区悬崖(放大器,已改减除式)**:残余一旦过线(0.06g)就吃全额增益 → ~21px/s 匀速爬。

**保留的修复 · 减除式死区**:只把超出死区的部分交给增益(`(tmag−死区)/tmag` 缩放),消除"过线即全额恒力"的悬崖——零点即使略有污染,残余小力也会被阻尼压住。**注意**:减除式让小倾斜有效力整体减小 0.06g,手感略"重";偏慢时微调 `TILT_GAIN`,勿动死区本身。

## 关卡库:16 张同难度真迷宫(2026-07-08 定案,取消难度渐进)

> 关卡演进史(4 关 → 8 关随机洗牌袋 → 12 关无岔路 → 16×12 分叉迷宫 12 关 → **16 张同难度**)见 git log;**以下为当前形态**。

- **动机**:幼儿实机体验——12 关版里 L1~L8(宽走廊/少岔路/短路径)已被完全掌握,洗牌轮换里混着太简单的图让孩子无聊。定案:**取消难度渐进(tier/guide 字段删除),16 张全部按原最难档(旧 Tier 3)之上出题,难度带收窄**。网格不变:`MAZE_COLS/ROWS = 16×12,MAZE_CELL = 20px`,物理手感参数一律未动。
- **统一难度带(离线脚本实测)**:最短路 **44~58 格**、**12~18 弯**、死胡同 **3~5 条**、环路 **1~2 个**;全部 **1 格窄走廊(20px)**。难度来源:分叉+死胡同(走错要折返)、环路(两条都通,给"选路"而非"对错")、长回旋主路。零失败原则不变(死胡同只是"还没到")。
- **出题方式(手工编排纪律不变)**:离线脚本(7×5 节点格回溯法挖迷宫 + 开 1~2 环墙 + 指标过滤)产候选,人工挑 16 张定案,再按拉丁方施加镜像/旋转让**家分布四角**(每世界 4 关各占一角,避免视觉重复);**运行时仍是静态数据表 + 进关 BFS 校验,无任何随机生成**。每关注释附指标与生成种子(maze.c)。
- **星规则(不变)**:星藏在**偏离最短路的死胡同底**引诱探索(星非过关必需,死胡同因此变成"寻宝"而非挫败);每关 1~2 颗。
- **过关切关 = 洗牌袋**(不变):一轮内 16 关各出现一次,打完重洗;**开局不再固定 L1**(同难度下无"最易关"),`first_level_from_bag` 直接随机起手。
- **家长菜单难度档已删**:同难度下"Easy=前 2 关"失义,`Levels` 按钮与 `game_state_set_level_band()` API 一并移除(家长菜单余:亮度/音量/震动/Home/Back)。
- **设计纪律 → 已入库 `tools/verify_mazes.py`**(直接解析 maze.c):①BFS 连通 + 全路面格可达(无孤岛);②S/H/* 网格字符与 start/home/stars 字段一致 + 边框完整;③报告死胡同/岔口/环路数与最短路长/弯数。当前 16 关全过。
- ✅ **实机验收通过(2026-07-08,含五角星精灵)**。若后续觉得整体偏难:先压 `VEL_MAX` 220→~160 降速,仍不行再用脚本按更短路/更少死胡同重出一批,**勿手改单格墙**(改完必跑 verify_mazes.py)。
