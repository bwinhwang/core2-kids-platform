## 0. 立项依据(为什么是它)

1. **载具图式**是 3 岁段最强主题偏好,"接送乘客"自带角色扮演叙事,重复游玩有内生动力;
2. **速度型控制是全平台空白技能轴**:tilt 是力型(倾斜=加速度)、旋钮是位置型(拧多少
   在哪),摇杆推多少车走多快是**速度型**——真正的新操控体验,不是三强换皮;
3. **决策**:乘客想去哪(图泡)→ 认颜色/认房子 → 开过去,认知匹配 + 空间导航双决策;
4. **收集元循环**:送达一个亮一栋房,全部送达全镇亮灯派对。

## 1. 复用资产(红线:能复用的不重写)

- **传输/绑定层**:`chain_bus` / `unit_chain_joystick` 零改动;`chain_lab.c` 的
  `scan_bus`/`poll_joy`/`node_rgb`/`hue2rgb`/`joy_calibrate_center` **整体形状照抄**
  (拷贝进本工程收窄为 joystick-only;chain_lab 本体不动)。20Hz 轮询节奏、
  `activity` 返回值喂 `core2_sleep_kick`、DEEP 唤醒重扫,全套沿用。
- **摇杆标定值**:`JOY_HALF_SPAN`/`JOY_DEADZONE`/`JOY_INVERT_X`(左右反已定案)等
  直接抄 chain_lab tuning 定案值(同一只摇杆)。
- **滑行碰撞**:tilt_maze 撞墙滑行(房子/树/喷泉 = 矩形/圆形障碍,撞上滑走不粘住)。
- **无单元提示卡 / 拔线容错 / 热插拔重扫**:busy_knobs→chain_lab 的通用形态照搬
  (提示卡 pictogram 换成摇杆+插头)。
- **家长菜单**:等 ROADMAP R2 `components/parent_menu` 组件化后统一接入,未就绪先不带。

## 2. 玩法总览

**单屏俯视小镇**(静态层一次画完):浅色环形马路(纯装饰,**不约束行驶**——满屏都能开,
零失败)、**三栋彩色房子**(红/蓝/绿,门口各有门垫区)分布三方、两棵树 + 中央小喷泉
(障碍物)、若干**站牌点**、左下角车库(巴士起点)、四周树篱围边。

摇杆开一辆**小巴士**自由行驶。站牌处等着小动物乘客,头顶**图泡**画着想去的房子
(房形 + 颜色双编码,无文字)。开到乘客身边 → 蹦上车(图泡移到车顶);开到对应房子
门垫 → 蹦下车进门,房子亮灯 + 窗口小脸挥手。三位乘客全部送达 → 全镇亮灯派对 →
新一批乘客。**摇杆按键 = 喇叭**(叭叭两声 + 车身弹一下;附近乘客会挥手——纯快乐,
无机制,复用"按键=快乐噪音"的已验证模式)。

## 3. 输入 → 机制映射

| 输入 | 来源 | 机制 |
|---|---|---|
| 摇杆 X/Y(归一化) | `poll_joy`(死区/归中/轴向抄 chain_lab 定案) | **速度向量**:`v = joy_vec × BUS_SPEED_MAX`,EMA 平滑;推多少走多快,松手即停(强"阻尼"由 EMA 提供) |
| 速度方向 | 由 v 算 atan2 | 车头朝向,量化到 **8 档预烘精灵**,带角度迟滞防边界抖动换图 |
| 摇杆按键 | `press_edge` | 喇叭(节流 `HONK_COOLDOWN_MS`) |
| 链上 encoder(若在) | — | v1 不用(§11⑤),节点灯设常暗氛围色 |

**刻意不做车头转向式驾驶**(前进+转弯的双自由度耦合):方向盘模型对 3~4 岁太难,
全向"推哪走哪"才是这个年龄段的正确映射(等同遥控玩具车的直觉)。

## 4. 状态机

```
[SCAN] ──(scan_bus 认领到 joystick)──► [ROUND_SETUP] ──► [PLAY]
   │ 没认到 → 无字提示卡(摇杆+插头图)+ 周期重扫(通用形态)

[ROUND_SETUP]  从站牌表随机选 3 点放乘客,目的地 = 三栋房各一(不重复);巴士回车库
[PLAY]         20Hz:poll_joy → 速度积分 → 障碍滑行 → 乘客/门垫判定 → 渲染 → 反馈
[PLAY] ──(3 位全送达)──► [PARTY](~3.5s 全镇亮灯+彩纸+SND_WIN+彩虹)──► [ROUND_SETUP]

每位乘客独立子状态:WAITING(站牌处待机小晃)→ BOARDING(蹦上车 ~350ms)→
RIDING(隐身,图泡挂车顶)→ ALIGHTING(蹦下车进门 ~350ms)→ HOME(房子亮,窗口小脸)

载客容量 = 1(v1):RIDING 期间碰到其他 WAITING 乘客不响应(乘客原地招手示意),
先送完再接——因果链一次只有一条,认知负荷可控(§11③)。
```

## 5. 核心机制细则

- **接客**:空车 && 巴士中心与乘客距离 < `PICKUP_RADIUS_PX` → BOARDING。判定用距离圆,
  不要求停稳、不要求对准(容差给够,开过去蹭到就算)。
- **送达**:载客 && 巴士进入**目的房门垫区**(门口 ~48×32 矩形)→ ALIGHTING。
- **错门零失败**:载客进入**非目的房**门垫 → 乘客探头摇头 + 轻"嗯?"上扬短音(中性)+
  该房窗口小脸眨一下("这家不是哦")——**不下车、不扣分、无负面音色**,带
  `WRONG_DOOR_SND_COOLDOWN_MS` 节流(门口来回蹭不机枪音)。
- **撞障碍**:滑行 + 轻 boing + 车身挤压脉冲(缩放一下);速度本来就封顶,无痛。
- **喇叭社交**:HONK 时 `HONK_WAVE_RADIUS_PX` 内的 WAITING 乘客挥手 + 短应答音——
  教会孩子"按喇叭大家理你",纯情绪奖励。
- **朝向精灵**:init 时把程序化画的巴士基图(车身圆角矩形 + 车窗 + 车灯)按 8 个朝向
  超采样旋转烘成 8 张 ARGB 图(付一次初始化成本,运行时只换图 + 挪坐标,零运行时
  旋转变换);朝向切换带 `HEADING_HYST_DEG` 迟滞,防 45° 边界来回抖动闪图。
- **动画所有权纪律**:BOARDING/ALIGHTING 走 `lv_anim`,完成判定用 game_task 侧帧
  倒计时,逻辑坐标存影子变量(busy_knobs 小鸟先例,不跨任务碰 LVGL 状态)。

## 6. 反馈矩阵(六类事件 × 五通道,摇杆节点 RGB 为第五通道)

| 事件 | 屏幕 | 音频 | 触觉 | 底座灯带 | 摇杆节点 RGB |
|---|---|---|---|---|---|
| 行驶 | 巴士脏矩形移动 | 静音(连续移动不配音) | — | — | 空车=暖白微亮 |

## 7. 渲染预算合规(根 `CLAUDE.md` §6)

- 静态层:小镇全景(马路/房子/树/喷泉/树篱/站牌)进场画一次;房子"亮灯"= 预画两态
  (暗窗/亮窗)切换,不做运行时发光;
- 动态层:巴士 28×28 脏矩形 ×2 ≈ **1.6k px/帧**(主角,60fps);乘客待机小晃走
  5–10fps 氛围档(各 ~14×16,三只合计远在预算内);图泡只在接/送时改;
- 8 张朝向精灵 = 8 × 28×28 ARGB ≈ 25KB,PSRAM 内毛毛雨;
- 派对彩纸限量 ≤8 片,15–20fps 庆祝档;全程零运行时 alpha 混合。

## 9. `tuning.h`(★ = 最可能要实机标定)


## 10. 电源 / 省电(桌面玩法,坑照 chain_lab)

- kick 判据 = `poll_joy` 的 `activity`(偏移/按键任一),现成;
- NAP:5V 还在,推摇杆可唤醒;DEEP:切 5V → 节点断电复位 → 唤醒重扫 + **巴士回车库、
  RIDING 乘客保持在车上、已送达进度不清**(仿 chain_lab `crane_game_reset_position`
  的"复位安全位置、不吞进度"原则);
- 拔线:连续 20 帧读失败 → 提示卡;插回重扫接管。

## 11. 开放风险(实机验证时决策)

1. **速度型控制对 3~4 岁的难度是本作最大未知**(P1 单独验证)。降级预案已想好:
   **恒速方向模式**——摇杆只取方向、速度恒定 `BUS_SPEED_MAX×0.6`,推=走松=停
   (做成 tuning 开关 `BUS_DRIVE_MODE`,一行切换,实机 A/B)。
2. **图泡可读性**:不识字前提下"房形+颜色"双编码是否 2 分钟自发理解(magic_wand
   教训的成功标准);不行加强:目的房屋顶同步慢闪呼应图泡。
3. **容量 1 vs 多**:v1 定死 1;实机若孩子明显想"接一车",容量 2+ 车顶多泡是第一个
   趣味增量批候选。
4. **8 朝向精灵观感**:量化换图是否显得"跳"(vs 平滑旋转);迟滞 + 车身近圆角设计
   已尽量弱化,不行加到 16 朝向(烘图翻倍,运行时代价不变)。
5. **encoder 节点闲置**:链上若挂着 encoder,v1 只设氛围灯;后续候选=车载音乐盒
   (转=换电台旋律)留趣味增量批。
6. **链路强度**:孩子会长时间把摇杆推到底,链路稳定性抽查沿用 chain_lab §11⑤ 结论,
   若 busy_bus 实测出现掉链,`chain_bus_sniff` 排障路径可用(诊断走 chain_lab 卡带,
   本作不内置诊断模式)。

## 12. 里程碑与验收(先验车感再盖楼)
