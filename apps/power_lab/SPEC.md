# power_lab SPEC —— 功耗/系统评估台

> **本文件是功能规格**(这个 app 该做什么);**竣工现状 / 落地差异 / 待实机点检见同目录
> `README.md`(as-built)**。实现前先读根 `CLAUDE.md` §2(六条设计原则)、§4(组件层)、
> §6(渲染红线)、§7(电源·休眠,**本 app 是省电纪律的明确例外**)、§8(UI 可读性)、
> §11.2(AXP192 电流测量的坑,重点)。占 `ota_5`(`0xB90000`),槽表见 `tools/flash_map.md`。

## 0. 定位

`power_lab` 是平台的**功耗/系统评估台**:评估对象不是外接单元,而是**这台机器自己**——
AXP192 遥测电池/VBUS 的电压电流、一套负载开关矩阵让人亲眼看见"打开这个功能,电流跳了多
少"、休眠演练验证 NAP/DEEP 两级省电确实生效、续航估算给出量级参考、SPIFFS 离线录制解决
"真实续航测试必须拔 USB → 没串口"这个根本矛盾。不是游戏,追求 CLAUDE.md §2 的六条评估台
原则,其中 §7 省电纪律本 app **主动接管**(不是继承默认行为)。

## 1. 覆盖的评估维度

| 维度 | 数据来源 | 组件 |
|---|---|---|
| 电池电压/电流(充/放)| AXP192 ADC | `power_monitor` |
| VBUS(USB)电压/电流/在位 | AXP192 ADC | `power_monitor` |
| 充电状态 | AXP192 REG 0x00 | `power_monitor` |
| 负载开关(背光/灯带/5V/喇叭/震动/CPU 频率)| 直控 AXP192/led/audio/haptic/esp_pm | `core2_power`/`ledstrip_fx`/`audio_fx`/`haptics`/`esp_pm` |
| 休眠演练(NAP/DEEP 手动触发)| `core2_sleep_force_stage` | `core2_sleep` |
| 续航估算 | `batt_discharge_ma` 线性外推 500mAh | `power_monitor` |
| 离线录制(拔 USB 场景)| SPIFFS 文件 | `data_log`(v2 rec_* API)|

## 2. 页面结构(320×240,顶 24px 状态栏常驻 + 中 192px 翻页内容 + 底 24px 翻页条)

### 2.1 Page 1:遥测面板 + 负载开关矩阵

- 左列(0,0,132,192):`ui_list_menu` 6 行——背光挡位(0/10/60/100%循环)、灯带亮度
  (0/48/255 循环)、5V EXTEN 开关(`with_switch=true`)、喇叭测试音(瞬时触发)、震动测试
  (瞬时触发)、CPU 锁频(240MHz 定频 / 80~240MHz 自动调频循环,`esp_pm_configure` 不可用
  时禁用态)。
- 右列(136,0,184,192):充放电状态文字行 + BattV/BattI 两张数值卡 + VBUS 电压/电流组合
  文字卡(未接时显式"未接")+ 电流趋势 chart(滑动平均,数据源在 VBUS 在位/不在位间自动
  切换,见 §11.2 已知坑)。
- 1Hz 轮询(power_monitor 的 AXP192 ADC 本身就是这个量级,不需要更快)。

### 2.2 Page 2:休眠演练 + 续航估算 + 离线录制

- 左列(0,0,132,192):"演练 NAP"/"演练 DEEP" 按钮、库仑计复位按钮、录制开始/停止切换
  按钮、Dump 导出按钮 + 演练状态文字行。
- 右列(136,0,184,192):回放均值电流 / 回放时长两张数值卡(演练结束后填充)、续航估算
  数值卡(充电中或电流太小时显式"充电中"/"N/A" 而非乱数)、录制状态文字行。

### 2.3 密度合规(CLAUDE.md §8)

Page 1 用到 2 张数值卡 + 1 张文字卡 + 1 个 chart;Page 2 用到 3 张数值卡。均未超过"单屏
最多 4 数值卡 + 1 chart"的密度上限(list_menu/按钮列表不计入该预算,是独立的控制通道)。

## 3. 负载开关矩阵设计要求

- 每个负载项的开关动作必须真实调用对应的硬件控制 API(不是只改 UI 状态),让电流表(实机
  用万用表/AXP192 遥测本身)能看到真实台阶。
- 背光 0% 档必须叠加 `core2_power_backlight(false)` 断 DCDC3 使能位,不能只调
  `bsp_display_brightness_set(0)`(CLAUDE.md §7 坑 3:那样屏幕仍有 ~2.95V 微光,电流台阶
  不明显)。
- CPU 锁频项查证优先级低于其它 P1~P3 核心功能;查证不顺可降级为两档开关或砍掉整项,不许
  硬凑不确定的实现(任务原始要求,已查证通过,见 README)。

## 4. 休眠演练设计要求

- 用 `core2_sleep_force_stage()` 手动触发 NAP/DEEP,不等待真实静止 12s/60s。
- 演练期间做电流采样(内存数组,不需要 UI 刷新——深睡时屏都黑了)。
- 演练结束(定时唤醒,不依赖真实动作检测)后在屏上回放这段时间的均值电流 + 持续时长。
- **不许 app 散装重拼 core2_sleep 内部固化的亮度→DCDC3→灯带→5V 顺序**——一律经
  `core2_sleep_force_stage()`。

## 5. 续航估算设计要求

用 `power_telemetry_t.batt_discharge_ma`(或 `coulomb_mah` 增量)+ 已知电池容量 500mAh
做简单线性外推(mAh / 放电电流 mA = 剩余小时数)。充电中或放电电流太小(≈0,含 ADC 噪声
残留)时不应给出误导性的巨大数值,应显式呈现"当前无法估算"而非裸数字。

## 6. 数据导出

- 串口 CSV(`data_log` v1 API):P1~P3 阶段实时评估时用,复用 `tools/serial_capture.py`。
- SPIFFS 离线录制(`data_log` v2 rec_* API,本轮新增):P4 阶段拔 USB 场景专用——录制期间
  按合理频率(不需要太高,几分钟到十几分钟的典型录制时长远不会撑爆 2.4MB 分区)写入关键
  遥测列;插回 USB 后手动触发 Dump,把整份文件吐回串口复用同一套哨兵提取逻辑。
- 首次挂载 SPIFFS 若需要格式化,UI 必须显式提示("初始化存储中…"一类文字),不能让用户
  以为程序卡死。

## 7. 电源 / 省电(CLAUDE.md §7 明确点名的例外)

- **不做默认闲置省电**:power_lab 本身就是评估功耗的工具,主循环不调用 `core2_sleep_feed`
  跑自动打盹/深度省电——那套逻辑会和本 app 的负载开关矩阵状态互相打架。
- `core2_sleep_t` 仍要初始化,只经 `core2_sleep_force_stage()` 在"演练"场景下手动驱动。
- 深度演练允许真的切 M-Bus 5V/断背光(与 unit_bench 相反)——power_lab 不接 PORT.A 外接
  单元,断 5V 只影响灯带,这正是演练要验证的效果本身。

## 8. 里程碑(对应实施计划 P1~P4)

- **P1**:只读遥测面板(1Hz),状态栏 + 数值卡 + chart。
- **P2**:负载开关矩阵(背光/灯带/5V/喇叭/震动/CPU 锁频)+ 电流 chart 能看出台阶。
- **P3**:休眠演练(NAP/DEEP 手动触发 + 采样回放)+ 续航估算。
- **P4**:SPIFFS 离线录制(启停按钮)+ Dump 回连导出按钮。

## 9. 明确不做(本轮范围外)

- per-rail 电流测量(AXP192 硬件本身不支持,只有电池路和 VBUS 路两个测量点,见
  CLAUDE.md §11.2)。
- 长期(数小时以上)连续 SPIFFS 录制的存储空间自动清理/轮转策略。
- 真正的自动 light sleep(CPU 锁频矩阵项刻意只做 DFS 频率切换,不开
  `light_sleep_enable`,见 README「CPU 锁频查证结论」)。
- 阈值告警变色——续航/电流的"正常范围"因使用场景而异,不硬编码默认阈值(与 unit_bench
  SPEC §9 同样的取舍)。
