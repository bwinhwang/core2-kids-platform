## 0. 立项依据(为什么是它)

对准三强归因的四条(`docs/ROADMAP.md` §0):

1. **实体感**:倾斜整机,全身本体感,与最受欢迎的 tilt_maze 同输入;
2. **连续比例 + 群体放大**:一个倾斜动作带动**一整群**(~10 只)小动物哗啦啦滑动——
   单球给不了的"权力感",也是与 tilt_maze 的第一差异轴;
3. **决策随技能成长**:两类动物分归两个家(小鸡→鸡窝、小鸭→池塘),要"先倒一边、再倒
   另一边";群体控制比单球更考验倾斜的轻柔度,倒猛了四散(搞笑场面,不是失败);
4. **收集元循环**:每只归家 = 五声音阶上行一音 + 家门口探头小脸计数;全部归家 = 派对。

**倾倒/归位是幼儿最强图式之一**(见什么倒什么、玩具要回家),语义匹配(动物↔家)比纯
颜色匹配更本能,颜色只做双重编码。

## 1. 复用资产(红线:能复用的不重写)

- **倾斜物理**:`physics.c` 的增益/阻尼/速度封顶/**减除式死区**/低通/轴映射,连同
  tilt_maze 的定案值起步(`TILT_INVERT_X=1` 等,同一台机器同一标定);扩成 N 对象循环。
- **滑行碰撞**:tilt_maze 撞墙滑行代码(边界围栏 + 圆形障碍)。
- **烘焙精灵**:init 时点内测试 + 4×4 超采样烘 ARGB 图(tilt_maze 五角星先例)——小鸡/
  小鸭各一张基础身体图,眼/喙用子对象叠(仿 chain_lab `PRIZE_LOOK` 装扮法)。
- **摇一摇检测**:busy_knobs 的泄漏计数 + 高阈值 + 冷却(`SHAKE_*` 三常量原样抄)。
- **省电**:`core2_sleep` 全托管。**纯 IMU 玩法没有桌面省电坑**——玩的时候机身必然在动,
  `core2_sleep_feed()` 喂加速度即可,无需任何 `core2_sleep_kick`(平台里最省心的一档)。
- **状态机骨架/家长菜单**:仿 tilt_maze `game_state.c`(ATTRACT/PLAY/WIN 三态 + 打盹
  托管);家长菜单等 ROADMAP R2 的 `components/parent_menu` 组件化落地后统一接入,
  组件未就绪则本卡带先不带菜单(与 busy_knobs 等现状一致),**不自己再写一份**。

## 2. 玩法总览

**单屏俯视后院**(静态层一次画完):四周木栅栏;**左边缘中段 = 红顶鸡窝**(门口朝右)、
**右边缘中段 = 蓝色池塘**(缺口朝左);四角各一丛**圆形灌木**(烘进静态层 + 碰撞圆,
把直角变斜坡,**防动物堆死在角落**——这是布局层面的防挫败设计,不是装饰)。

场中散落 **10 只小动物:5 只黄小鸡 + 5 只白小鸭**(半径 ~8px 圆身 + 眼/喙)。倾斜整机,
全体顺"重力"滑动;把小鸡倒进鸡窝、小鸭倒进池塘。**家门是单向门**:进去就不出来
(进度只增不减);进错家 = 被温柔弹开 + 摇头 + 轻"啵"(还没到,不是错)。
10 只全归家 → 派对 → 重新散一批 → 循环。

## 3. 输入 → 机制映射

| 输入 | 来源 | 机制 |
|---|---|---|
| 倾斜向量(x/y) | `imu_mpu6886` + tilt_maze 滤波链 | 全体动物的加速度方向与大小(连续比例) |
| 每只动物的增益抖动 | init 时随机 ±`GAIN_JITTER_PCT`% | 同一倾斜下各自快慢略不同 → 群体自然散开,不结成一坨(有机感的关键) |
| 摇一摇 | busy_knobs 泄漏计数法 | 彩蛋:在场动物全体原地小跳 + 叽嘎合唱 + 灯带彩虹一闪;**不改进度** |
| 触摸屏 | — | v1 不用(开放风险 §11⑤) |

## 4. 状态机

```
[ATTRACT] 动物们原地睡(Zzz + 慢呼吸缩放),倾斜超 ATTRACT_TILT_THRESH
   │        → 全体醒来小跳 + 合唱一声 → [PLAY]
[PLAY]    60Hz:读倾斜 → N 对象物理(§5)→ 门判定 → 渲染脏矩形 → 反馈事件
   │        任一动物进对的家 → 捕获动画(§5.3),计数 +1
[PLAY] ──(10 只全归家)──► [PARTY](~3.5s:两家一起弹跳 + 限量彩纸 + SND_WIN
   │                        + HAPTIC_WIN + 灯带彩虹)
[PARTY] ──► [SCATTER](重散一批:约束随机布点,§7)──► [PLAY]

打盹/深度省电:core2_sleep 全托管,任一状态可进出(同 tilt_maze,唤醒回当前进度,
进度不清零——归家的不放出来)。
```

## 5. 核心机制细则

### 5.1 群体物理(每帧,60Hz)

1. 每只:`acc = tilt_vec × TILT_GAIN × (1 + jitter_i)` → 阻尼 → 封顶(全套 tilt_maze);
2. **两两软分离**:距离 < `2×ANIMAL_R + SEP_PAD` 时沿连线做位置修正(各推一半),
   10 只 = 45 对检查,O(N²) 在此规模可忽略;**不做真刚体碰撞**,软推挤即可(挤挤挨挨
   本身是可爱的);
3. 栅栏/灌木/家的外墙:滑行碰撞(不粘住);
4. 门区判定(§5.3)。

### 5.2 门(单向 + 语义匹配)

- 每家门口一个**门区矩形**(比动物直径宽 ≥2.5 倍,容差给够);
- 动物中心进入门区:**种类匹配** → 捕获;**不匹配** → 沿门法线方向温柔弹出
  (`GATE_BOUNCE_SPEED`)+ 摇头动画(左右各摆一下)+ 轻"啵"音 + `HAPTIC_BUMP_LIGHT`。
  同一只动物的弹出反馈带 `BOUNCE_SND_COOLDOWN_MS` 节流(倾斜顶着门蹭不会机枪音)。

### 5.3 捕获(所有权交接,防跨任务踩状态)

命中即**从物理仿真中移除**(`active=false`),精灵交给一条 `lv_anim` 演"蹦进门"
(~350ms,位移 + 缩小),完成判定用 game_task 侧帧倒计时(**仿 busy_knobs 小鸟先例:
逻辑坐标存影子变量,不反查 LVGL 对象、不用 anim ready 回调跨任务碰状态**)。
同时:家门口探头小脸 +1(计数显示,鸡窝窗口冒鸡头/池塘水面冒鸭头)、
叽/嘎短鸣 + **五声音阶第 n 音**(n = 已归家总数 1..10,进度天然可听)、
`HAPTIC_COLLECT`、灯带 `LED_FX_COLLECT` 扫圈;归家数 ≥8 时灯带基色加亮一档(冲刺感)。

### 5.4 重散布点(约束随机 + 加载校验)

平台纪律"内容手工编排、别纯随机"(根 CLAUDE.md §11)针对的是**可解性**(迷宫);
本作是开阔场地、无可解性问题,**刻意选择约束随机**换每轮新鲜感,校验兜底:
布点约束 = 两两间距 ≥`SCATTER_MIN_GAP`、离两家门区 ≥`SCATTER_GATE_CLEAR`、避开灌木、
在栅栏内;拒绝采样 `SCATTER_MAX_TRIES` 次不满足则退化为预置网格点(永不失败)。
布完跑一遍校验断言(点数/间距/越界),失败走预置网格 —— 对应 verify 纪律。

## 6. 反馈矩阵(五类事件 × 四通道)

| 事件 | 屏幕 | 音频 | 触觉 | 底座灯带 |
|---|---|---|---|---|
| 醒来(ATTRACT→PLAY) | 全体小跳 | 合唱短鸣(叽×2嘎×1) | `HAPTIC_HELLO` | AMBIENT 暖亮 |
| 撞栅栏/灌木(速度超阈) | 挤压一下(缩放脉冲) | 极轻 boing(节流) | —(群体撞墙高频,不震) | — |
| 进错家弹出 | 弹出 + 摇头 | 轻"啵"(中性,不低沉) | `HAPTIC_BUMP_LIGHT` | — |
| 归家 | 蹦进门 + 探头小脸 +1 | 叽/嘎 + 五声音阶第 n 音 | `HAPTIC_COLLECT` | `LED_FX_COLLECT`;≥8 只基色加亮 |
| 全归家派对 | 两家弹跳 + 限量彩纸(≤8 片,15–20fps) | `SND_WIN` + 上行琶音 | `HAPTIC_WIN` | `LED_FX_WIN` 彩虹 |
| 摇一摇彩蛋 | 全体原地小跳 | 叽嘎合唱 | `HAPTIC_BUMP_MED` | 彩虹一闪 |

## 7. 渲染预算合规(根 `CLAUDE.md` §6)

- 静态层:后院地面/栅栏/鸡窝/池塘/四角灌木,进场画一次;
- 动态层:10 只动物 ≈ 18×18 脏矩形 ×2(旧+新)×10 ≈ **6.5k px/帧**,预算(15k)内富余;
  捕获动画/摇头/挤压都是单精灵短脉冲,不叠加超预算;
- 探头小脸计数:归家时改一次,不逐帧;
- 派对彩纸限量 ≤8 片,走 §6.5 庆祝档(15–20fps 可接受);
- 全程不透明色块 + init 烘焙精灵,零运行时 alpha。

## 9. `tuning.h`(默认值 = tilt_maze 定案值起步,★ = 最可能要实机标定)

```c
// ── 群体 ──────────────────────────────────────────────
#define ANIMAL_COUNT        10     // ★ 视觉混乱就减到 8
#define ANIMAL_KINDS        2      // ★ 分拣认知过难可降 1(全归一家),过易升 3
#define ANIMAL_R            8
#define GAIN_JITTER_PCT     12     // 每只增益抖动 ±%,群体散开的关键
#define SEP_PAD             2      // 软分离触发间距余量
#define SEP_CORRECT_PCT     35     // ★ 每帧位置修正比例(太大抖、太小叠)

// ── 手感(tilt_maze 定案值起步)─────────────────────────
#define TILT_GAIN           1800
#define DAMPING             0.90f
#define VEL_MAX             180    // 低于 maze 的 220:群体可读性优先
#define DEADZONE            0.06f  // 减除式,勿动(tilt_maze 定案)
#define TILT_ALPHA          0.25f

// ── 门 / 布点 ─────────────────────────────────────────
#define GATE_W              44     // 门区宽 ≈ 动物直径×2.75
#define GATE_BOUNCE_SPEED   70
#define BOUNCE_SND_COOLDOWN_MS 300
#define CORNER_BUSH_R       36
#define SCATTER_MIN_GAP     24
#define SCATTER_GATE_CLEAR  40
#define SCATTER_MAX_TRIES   200

// ── 节奏 / 彩蛋(busy_knobs 定案值起步)──────────────────
#define CAPTURE_ANIM_MS     350
#define PARTY_HOLD_MS       3500
#define ATTRACT_TILT_THRESH 0.22f
#define SHAKE_THRESH        1.2f
#define SHAKE_NEEDED        3
#define SHAKE_COOLDOWN_MS   2000
```

## 10. 电源 / 省电

纯 IMU 玩法 = 平台最简省电形态:每帧 `core2_sleep_feed()`,无 kick、无单元断电重接管。
打盹(12s)/深度省电(60s)/去抖唤醒全部默认托管;唤醒回当前进度。

## 11. 开放风险(实机验证时决策)

1. **群体手感是本作成败核心**(P1 单独验证):软分离参数不当会"结坨"或"抖动不安";
   增益抖动太小群体像一块砖、太大像没规律。
2. **分拣认知负荷**:3~4 岁对"两类分两家"是否即刻可懂——成功标准同 magic_wand 教训:
   **无人讲解 2 分钟内自发理解**;不行先降 `ANIMAL_KINDS=1` 保底(纯归家也成立)。
3. **最后一只的挫败**:四角灌木 + 软分离理论上防堆死,实机要专门试"最后一只在门口
   反复蹭"的场景;不行就加"最后一只自带轻微向家漂移"的隐形辅助(仅剩 1 只时生效)。
4. **与 tilt_maze 的差异化靠美术拉开**:俯视后院 vs 迷宫走廊,配色/音色刻意区分。
5. **触摸戳一下动物让它跳**(v1 不做):留作趣味增量批候选,防 v1 范围蔓延。

## 12. 里程碑与验收(先验手感再盖楼,magic_wand 教训)

**P1 群体手感验证(先烧这个)**:静态后院 + 10 只动物 + 倾斜物理 + 软分离 + 栅栏/灌木
滑行碰撞 + 撞墙音。**验收:倾斜赶群"重而稳不飘",群体散开自然、不结坨不抖;孩子表现出
反复倾倒的兴趣。** 不过关先调参,再不过关回设计层,不进 P2。
**P2 归家闭环**:两家 + 门判定 + 捕获/弹出 + 计数 + 音阶进度 + 派对 + 重散。
**P3 打磨**:ATTRACT 睡醒、摇一摇彩蛋、探头小脸细节、launcher 专属图标(小鸡+窝,要重刷
launcher)、flash_one.sh/flash_map.md 登记、README as-built、根 CLAUDE.md §1/§12 索引更新。
