## 与 SPEC 的落地差异(均有理由)

1. **家可读性加强批(2026-07-12 实机反馈"小鸡的家不够明显",SPEC 外新增)**:
   大拱门(奶油门框 + 深棕门洞,跨骑门区全高,原 6px 细条门洞否决)、门口沙色**门垫**
   (伸进场地,与池塘浅滩缺口对仗的"从这儿进"地面邀请)、两家门口**住户脸招牌**
   (黄小鸡/白小鸭大圆脸,不识字 → 语义靠脸,兼防 §11② 分拣认知卡壳)。
2. **探头小脸挪到屋顶天窗条**:SPEC §5.3 原文"鸡窝窗口冒鸡头"在屋身;屋身让位给大门
   + 招牌后挪上屋顶(池塘侧不变,仍在水面高光带下方)。
3. **撞墙音节流是全群共用 250ms 窗口**(`WALL_BUMP_SND_COOLDOWN_MS`,SPEC 只说"节流"):
   10 只可能同帧蹭栅栏,按只节流仍会机枪音;挤压视觉不节流(每只自己的因果)。
4. **`SHAKE_COOLDOWN_MS 2000`**(SPEC 值;busy_knobs 实际定案 1500,本作彩蛋音效更长取 2000)。
5. **ATTRACT 醒来判定** = 水平两轴合成幅度 `sqrt(ax²+ay²) > 0.22g`(与屏轴映射无关,平放噪声 ~0.02g)。
6. **SCATTER 不设独立状态**:重散是 PARTY 倒计时归零时的一次性动作(散点约束随机 + 拒绝
   采样 + 校验断言,失败退预置网格,永不失败)。

## 实现要点(跨任务纪律,仿平台先例)

- **捕获所有权交接**(§5.3):`flock_step` 置 `active=false` + 事件 → game_task **即时计数**
  (不等动画)→ `critters_capture` 起 lv_anim"蹦进门"(直线插值 + 抛物弧 + 缩到 44%),
  完成回调只隐藏自己的 LVGL 对象,不碰游戏状态(busy_knobs 小鸟影子变量纪律)。
- **脉冲三兄弟**(挤压/摇头/小跳):反馈侧只写计数,`critters_update` 在 game_task 帧循环里
  推进;摇头用 translate_x、小跳用 translate_y,与 `set_pos` 互不打架。
- **弹出节流**:物理每次照弹(flock),反馈每只 300ms(game_state 时戳)。
- **省电**:core2_sleep 全托管(纯 IMU 玩法无 kick);PARTY 态 `nap_eligible=false`;
  ATTRACT/PLAY 可打盹,唤醒回当前进度(归家的不放出来)。
- **音阶进度**:归家第 n 只播五声音阶(C 宫)第 n 音 + 叽(高双音)/嘎(低双音)前缀,
  单笔 `audio_fx_play_notes`(~290ms < 400ms 合成上限)。

## 已知小项

- 深度省电唤醒后灯带基色回 AMBIENT,归家 ≥8 只的冲刺加亮被复位(下次归家即恢复,不修)。
- 家长菜单未接(等 ROADMAP R2 `parent_menu` 组件化统一接入,与 busy_knobs 现状一致);
  游玩中无回 launcher 软入口,复位/断电回 launcher。
- 睡着时拿起:先亮屏(core2_sleep 唤醒)、再倾斜超阈动物才醒,两段式,预期行为。
